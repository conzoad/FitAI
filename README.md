# КБЖУ Трекер — Фитнес Приложение с ИИ

Android приложение на React Native (Expo) для отслеживания калорий и КБЖУ с интеграцией Gemini AI.

## Возможности

- 📝 **Добавление еды двумя способами:**
  - Текстовое описание (пример: "куриная грудка 200г с рисом")
  - Фотография из камеры/галереи

- 🤖 **Gemini ИИ для анализа:**
  - Распознавание продуктов и подсчёт КБЖУ
  - Персональный фитнес-чат с учётом ваших целей и текущего прогресса

- 📊 **Отслеживание прогресса:**
  - Дневной дневник питания с группировкой по приёмам пищи
  - Кольцо прогресса калорий и круги макронутриентов
  - Статистика за неделю/месяц с графиками

- ⚙️ **Автоматический расчёт норм:**
  - Формула Mifflin-St Jeor для BMR
  - Учёт уровня активности и целей (похудение/поддержание/набор)
  - Автоматический расчёт БЖУ с учётом калорийности

## Предварительные требования

- Node.js 18+ и npm
- Expo CLI: `npm install -g expo-cli` (опционально)
- Gemini API ключ (https://ai.google.dev)
- Android устройство или эмулятор

## Установка

1. **Клонируйте репозиторий:**
   ```bash
   cd ~/Documents/project
   ```

2. **Установите Gemini API ключ:**
   Создайте файл `.env` в корне проекта:
   ```env
   GEMINI_API_KEY=your_api_key_here
   ```

   Получить ключ: https://aistudio.google.com/app/apikey

3. **Зависимости уже установлены** (см. `package.json`), но можете обновить:
   ```bash
   npm install
   ```

## Запуск приложения

```bash
npx expo start
```

Затем в терминале:
- Нажмите `a` для открытия на Android эмуляторе
- Или отсканируйте QR-код в Expo Go на физическом устройстве

## Первый запуск

1. Приложение откроет профиль пользователя (онбординг)
2. Заполните:
   - Имя
   - Пол, возраст, рост, вес
   - Цель (похудение/поддержание/набор)
   - Уровень активности
3. Нажмите "Начать" — приложение рассчитает вашу дневную норму
4. На вкладке "+" добавьте первый приём пищи

## Номальное использование

### Добавление еды

**Текстом:**
1. Перейдите на вкладку "+"
2. Выберите тип приёма (завтрак/обед/ужин/перекус)
3. Напишите описание блюда
4. Нажмите "Распознать"
5. Проверьте результат и сохраните

**По фото:**
1. Перейдите на вкладку "+"
2. Выберите "Фото"
3. Сделайте снимок блюда
4. Нажмите "Распознать"
5. Сохраните в дневник

### Просмотр прогресса

- **Главная:** кольцо калорий + макро за сегодня + последние приёмы
- **Дневник:** все приёмы с группировкой по количеству пищи
- **Статистика:** графики за неделю/месяц
- **Чат:** задайте вопрос ИИ — получите персональный совет

## Особенности реализации

### Gemini API интеграция

- **Модель:** `gemini-3-flash` (быстрая, мультимодальная)
- **Текстовый анализ:** промпт-нутрициолог возвращает JSON с КБЖУ
- **Фото-анализ:** base64 изображение + промпт → JSON с распознанными продуктами
- **Чат:** системный промпт с данными профиля + история последних 20 сообщений

### Состояние и персистентность

- **Zustand + AsyncStorage:** локальное хранилище всех данных (профиль, дневник, чат)
- Данные сохраняются автоматически при каждом изменении
- При переустановке приложения данные сохраняются (до 6 МБ на Android)

### Расчёты питания

- **BMR:** Формула Mifflin-St Jeor (зависит от пола, возраста, роста, веса)
- **TDEE:** BMR × коэффициент активности (1.2–1.9)
- **Цель:** TDEE ± корректировка (±500 ккал при похудении/наборе)
- **Макросы:** 30% белки, 25% жиры, 45% углеводы

### Языковизация

- Весь UI на русском языке
- Дата-форматирование с русской локалью (Пн, Вт, 15 февраля 2026)
- Промпты для Gemini на русском

## Обновление и расширение

### Добавить новый экран

1. Создайте файл в `src/screens/MyScreen.tsx`
2. Импортируйте в `src/navigation/RootNavigator.tsx`
3. Добавьте в Tab.Navigator или Stack.Navigator

- Палитра цветов: `src/theme/colors.ts`
- Шрифты: `src/theme/typography.ts`

1. Измените `FITNESS_CHAT_SYSTEM_PROMPT` в `src/services/prompts.ts`
2. Тестируйте на вкладке "Чат"

## Gemini API квоты и ограничения

**Бесплатный тариф Gemini API** имеет лимиты:
- 15 запросов в минуту
- 1500 запросов в день
- 1 миллион токенов в месяц

### Ошибка "Quota Exceeded" (429)

Если вы видите ошибку:
```
Превышен лимит запросов к Gemini API.
Подождите XX секунд и попробуйте снова.
```

**Решения:**

1. **Подождите указанное время** (обычно ~60 секунд)
2. **Проверьте квоту** на https://ai.google.dev/rate-limits
3. **Создайте новый API ключ** если достигли дневного лимита
4. **Перейдите на платный план** для больших объёмов

**Мониторинг использования:** https://ai.dev/rate-limit

**Советы по экономии квоты:**
- Не анализируйте одно и то же блюдо несколько раз
- Используйте текстовый ввод вместо фото (меньше токенов)
- Не отправляйте слишком длинные сообщения в чате
- В настройках можно сменить модель на `gemini-1.5-flash` (меньше лимиты, но быстрее)

## Известные проблемы

- LF/CRLF предупреждения при коммите (Windows) — не влияют на работу
- Gemini API иногда может распознавать блюда неточно — показывается confidence level
